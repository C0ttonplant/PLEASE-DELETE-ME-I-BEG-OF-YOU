Unit Tests

What is a unit test?

Unit tests are functions that test the input and output of a function. The most simple tests simply require giving the function being tested its inputs, and compare the output with the expected value.

This style of testing encurages functions to be pure, not using or altering the state outside of its scope. This style of programming is often refered to as MIT style. (and is personally my ideal method of programming)

Example 

                    
function mult(int a, int b) int {
    return a * b;
}
function test(any result, any expected) {
    if (result == expected) {
        print("test passed");
    } else {
        print("test failed");
    }
}
test(mult(1, 1), 1); // test passed
test(mult(1, 2), 2); // test passed
test(mult(2, 2), 4); // test passed
test(mult(3, 2), 6); // test passed

Problems

In an ideal world, all functions would be like this. even in the case of complex inputs and outputs, you would have the guarentee that the same inputs always give the same output. But unfortunatly this isnt the case, functions can rely on global variables, and even set those variables instead of returning all its effects.

Another issue that can be run into, are functions that require a difficult to reproduce context in a testing environment, for example a function that needs a window context, or a function that needs an application context.

Example 

                    
bool globalvariable = true;
bool isEqual = false;

function impure(int a, int b) int {
    globalvariable = a == b && !globalvariable;
    isEqual = a == b;
    return globalvariable;
}
function test(any result, any expected) {
    if (result == expected) {
        print("test passed");
    } else {
        print("test failed");
    }
}
test(impure(1, 1), false); // test passed
test(impure(1, 1), false); // test failed
test(impure(2, 3), false); // test passed
test(impure(2, 3), false); // test failed

Creating tests

In accordance with these issues, there are good methods to use when creating tests. First, consider the bounderies of the function, whats the largest possible number, what about the most negitive number. For functions that take strings, consider testing with extreemly large strings, empty strings, and even null pointers. Users can and will always give inputs outside your wildest imagination.

Second, review the function, and see how all the inputs get manipulated, If its neccisary for the function to set external variables, then simply add tests for how those variables are set. If a function relies on data from a database, or on external variables, creating a function to seed and reset the data will allow for easy testing. this function would also be a good place to create the neccicary context, for example a dummy window context.

Example 

                    
bool globalvariable = true;
bool isEqual = false;

function impure(int a, int b) int {
    globalvariable = a == b && !globalvariable;
    isEqual = a == b;
    return globalvariable;
}
function reset() {
    globalvariable = true;
    isEqual = false;
}
function test(any result, any expected) {
    if (result == expected) {
        print("test passed");
    } else {
        print("test failed");
    }
}
reset();
impure(1, 1);
test(isEqual, true); // test passed
reset();
impure(1, 1);
test(isEqual, true); // test passed
reset();
impure(3, 2);
test(isEqual, false); // test passed
reset();
impure(3, 2);
test(isEqual, false); // test passed

Automated testing

Sometimes its ideal to write automatic testing, where the code can test thousands of combinations incredibly quickly, using loops and random data. Or a new form of automatic testing, using AI to generate tests, and verify functions for you.

Using loops is ideal for functions that have lots of variation in their inputs, where a large number of values can be quickly tested. On the other hand, AI can be useful for writing tedious unit tests that cant be automated. For example, functions that take objects as input.

Example 

                    
function mult(int a, int b) int {
    return a * b;
}
function test(any result, any expected) {
    if (result == expected) {
        print("test passed");
    } else {
        print("test failed");
    }
}

for(int i = 0: i < 1000; i++) {
    int a = rand();
    int b = rand();
    print(a, b); // make sure to print the inputs, so the failing case can be known
    test(mult(a, b), a * b);
}

AI

Using AI to write unit tests is a new method for good unit testing. Using AI, its easy to write many unit tests, without the tedious friction of writing the same code over and over again. AI can also be good at making unexpected input data, just like how a user would. This allows for even more comprehensive testing. AI is also good for creating seeded data for nessicary databases and variables.

AI does have issues though, AI is good at generating the tests, but its not always good at generating the expected result. For example, a function that counts the number of times a char (r) appears in a string (strawberry) many AIs will give the incorrect result as well.

Limitations

Unit testing is good for individual functions, using unit tests can ensure your functions are properly implemented. But unit tests cant cover how functions interact with eachother, or how a full stack application will behave. Unit tests also cant cover bugs like race conditions, memory issues, or performance bottlenecks. It doesnt matter how perfect a function is, if you use its outputs wrong.

These situations can be covered better by other testing methods, which I am excited to hear about from my classmates.

Frameworks

While its totally possible to write the testing function yourself, using a testing framework allowes for much greater reporting, along with easier to produce tests. I will go into PyTest specifically, how to set up and use.

pytest is installed as a pip module, then can be run in the command line as `pytest` this will run all files with'*_test.py test_*.py'. Inside the test file there can be multiple ways of structuring the tests, but i will go over how I find it the neatest

Example test_main.py 

                    
import pytest
# import the functions to test
from examples import mult, count

# create the test class, can be any name that starts with `Test`
class TestClass:

    # create test functions, can be any name that starts with 'test_'. paramater `self` returns `None`
    def test_mult(self) -> None:

        # assert is used as the test, this value should always resolve to a bool.
        assert mult(1, 2) == 2

        # multiple asserts can be in the same function
        assert mult(3, 4) == 12

        # for impricise floats, `approx` can be used to compair the near value
        assert mult(0.1, 0.3) == pytest.approx(0.03)

    def test_count(self) -> None:
        assert count('r', "straberry") == 3
        assert count('a', "apple") == 1
